Workflow Examples

Real scenarios and how to handle them:

Starting a New Feature

You're about to start working on a new feature and want to keep your work separate from main.

"Create a new branch called feature/user-authentication and switch to it"

Saving Your Progress

You've made some changes and want to save them with a descriptive message.

"Commit my changes with the message 'Add login form validation'"

Sharing Your Work

Your feature is ready for review and you want to create a pull request.

"Push this branch and create a PR to main"

Handling Conflicts

Someone else made changes to the same files you're working on.

"Pull the latest from main and help me resolve any conflicts"
Claude Translates for You

You don't need to know the exact Git commands. Claude Code understands your intent and executes the right commands. Focus on what you want to accomplish, not how Git works.

What You Say vs. What Git Does

You Say Git Command (for reference)
"Start a new branch for this feature" git checkout -b feature-name
"Save my changes" git add . && git commit -m "..."
"Upload this to GitHub" git push origin branch-name
"Get the latest changes from the team" git pull origin main
"I'm done, create a review request" gh pr create
