Build Your Daily Briefing Skill

Build a skill that synthesizes your meetings, communications, and documents into actionable intelligence — using nothing but Claude chat

Learning Objective

By the end of this module, you'll have a working .skill file that delivers a personalized daily briefing, pulling live data from Slack, Gmail, Calendar, Drive, and Granola meeting notes. You'll build it entirely in Claude chat and install it in your Claude account.

What You're Building

When you say "let's start our day" in any Claude conversation, your daily briefing skill will automatically read your context, search your connected systems, cross-reference everything, and present a structured briefing with clear next actions.

This isn't theoretical — you'll build, install, and run a real briefing during this module.

You Don't Need to Code Anything

You describe what you want in a Claude chat conversation. Claude builds the skill files for you. You download a .skill file and upload it to your Claude account. That's the entire process.

Download Full Guide (.md)

Prefer a hands-on approach? Download this guide, upload it to a Claude chat, and let Claude walk you through it interactively.

What's Inside a Skill

Your daily briefing skill has exactly two files:

Component What It Does You Provide Claude Handles
SKILL.md Instructions Claude follows when triggered Your data sources, briefing sections, alert logic Format, error handling, trigger phrases
references/context.md Your consolidated knowledge base Your role, team, OKRs, Slack channels, key docs Structure, organization, fallback data
daily-briefing/ ├── SKILL.md ← Instructions (what Claude does) └── references/ └── context.md ← Your knowledge base (who you are)

Why Skills, Not Folders
An earlier version of this training recommended building a folder structure in Cowork with a CLAUDE.md and multiple context files. The Skills approach is better:

Cowork Folder Approach Claude Skill Approach
File system resets between sessions Skill persists in your Claude account
Multiple files to maintain (profile.md, team.md, projects.md, etc.) Single consolidated context.md
Only works in Cowork desktop app Works in any Claude interface — web, mobile, desktop
Can't easily share or move your setup Portable .skill file
No built-in error handling Error handling in the skill instructions
Manual setup: create folders, edit files Claude builds it in a chat conversation

Before You Start

Prerequisites: MCP Connectors

Make sure you have these connectors enabled in Claude:

  • ✓ Slack
  • ✓ Gmail
  • ✓ Google Calendar
  • ✓ Google Drive
  • ✓ Granola (recommended — works with Zoom, Teams, Google Meet)

Where to check: Settings → Connected Apps (claude.ai) or Settings → Connectors (Cowork/Desktop)

Step 2: Build Your Skill in Claude Chat

Open a new conversation in claude.ai or the Claude app. Paste the following prompt, filling in your information from Step 1.

build prompt
I want to build a personal daily briefing skill for Claude. Here's my information: [Paste your completed worksheet from Step 1] Please build this as a .skill file with: 1. SKILL.md — the briefing instructions Claude follows when I say "let's start our day" 2. references/context.md — my consolidated knowledge base Important requirements: - Work with MCP connectors: Slack, Gmail, Calendar, Drive, Granola - If any data source fails, continue with the remaining sources - Include error handling (especially Granola, which can be flaky) - Google Docs use google_drive_fetch; Sheets need Chrome browser tools - Cross-reference signals across systems, don't just list them - End with "Over to you" and contextual follow-up prompts - Include day-specific logic (Monday OKR reminders, heavy meeting day prep, Friday wrap-up) Package it as a downloadable .skill file when done.
What Happens Next
  • Claude asks clarifying questions (e.g., "Which Slack channels are highest priority?")
  • Creates the SKILL.md with your customized briefing flow
  • Creates references/context.md with your consolidated knowledge base
  • Validates the skill format
  • Packages and presents a downloadable .skill file

This takes one conversation — usually 10-20 minutes.

Step 3: Install Your Skill

Three steps:

  1. Download the .skill file Claude created (appears as a downloadable file in the chat)
  2. Go to claude.ai → Settings → Skills
  3. Upload your .skill file

The skill is now active across all your conversations. No additional setup needed.

How to Update Your Skill Later

Start a new Claude chat, upload your current .skill file (or the individual SKILL.md and context.md), describe what you want to change, and ask Claude to rebuild. Then re-upload the new .skill file to replace the old one.

Step 4: Run Your First Briefing

Open a new conversation (not the one you used to build). Type:

prompt
Let's start our day

Claude will read your skill, load your context, gather data from all connected systems, cross-reference signals, and present your structured briefing. First run takes 30-60 seconds.

What to Evaluate

  • ✓Did it pull my calendar correctly?
  • ✓Did it find relevant Slack messages?
  • ✓Are the "Needs Your Attention" items actually urgent?
  • ✓Is the meeting prep useful?
  • ✗What's missing that I wish it had?
  • ✗What's noise I don't need?
  • ✗Any sections consistently empty?

Customize for Your Role

Not every section applies to every role. Adjust based on how you work:

Individual Contributors

  • Remove "Team Pulse" and "Team OKR Status" sections
  • Add project-specific tracking (e.g., "status of my PRs," "my open tickets")
  • Focus "Needs Your Attention" on your own deadlines and deliverables

Managers

  • Keep all team sections — "Team Pulse" and "Team OKR Snapshot" are essential for 1:1s
  • Add "1:1 prep" as a prominent follow-up prompt
  • Monitor direct reports' Slack channels for mood and blockers

Customer-Facing Roles (Sales, CS, Support)

  • Add deal/account channel monitoring
  • Include customer health signals from Slack
  • Track renewal or expansion activity
  • Add "deal intel [account]" as a follow-up prompt

Executives

  • Add competitive intel to the main briefing (not just as follow-up)
  • Include company-wide metrics and announcements
  • Monitor board-relevant items

What Makes a Great Briefing Skill

Aspect Good Great
Data sources Lists data from each system separately Cross-references signals across systems
Priorities Shows everything Filters to 48-hour action window
Error handling Breaks when a source fails Continues gracefully, notes what's missing
OKR data Hardcoded in the skill Fetched live from Drive with fallback
Meeting prep Lists meetings Pulls context from past meetings + action items
Alert logic None Manager repeat detection, deadline warnings, cross-source confirmation
Cross-System Intelligence

The real power isn't listing data — it's connecting signals:

  • Topic in Slack AND on today's agenda → flag in Meeting Prep
  • Action item from meeting connects to today's calendar → surface in Needs Your Attention
  • Doc comment AND meeting note reference same deliverable → high priority
  • Manager mentions something twice → they're tracking it, address proactively
  • Same blocker from someone 2+ times → needs escalation

Your skill instructions should tell Claude to look for these patterns.

Google Docs vs. Google Sheets

This distinction matters for how your skill fetches data:

Type How Claude Fetches It What Your Skill Needs
Google Docs google_drive_fetch — fast, direct MCP call The Doc ID (string between /d/ and /edit in URL)
Google Sheets Chrome browser tools — navigates to URL The full URL including tab/gid parameter

Tip: If possible, move critical data from Sheets to Docs — Docs are faster and more reliable since they don't need browser automation.

Troubleshooting

Problem Cause Fix
Briefing didn't trigger Skill not installed or wrong trigger phrase Check Settings → Skills. Try "let's start our day" exactly. Start a new conversation.
"Granola unavailable" warning MCP connector flaky or UUID error Normal — briefing continues without meeting data. Retry next time.
Missing Slack messages Channel names don't match or connector not enabled Verify exact channel names. Check Settings → Connected Apps.
OKR section shows "stale" warning Google Drive fetch failed Check Doc ID in context.md. Open doc directly to verify access.
Google Sheets data missing google_drive_fetch doesn't work with Sheets Sheets need Chrome browser tools — note this in your skill.
"Response could not be fully generated" Too much data, ran out of context Reduce Slack channels. Simplify less critical sections.

Key Takeaways

  1. Build in Claude chat — describe what you want, Claude builds the files. No Cowork, no code, no terminal.
  2. Two files is all you need — SKILL.md (instructions) and context.md (your knowledge base).
  3. Install once, use everywhere — your .skill file works across web, mobile, and desktop.
  4. Cross-system intelligence is the differentiator — connecting signals across Slack, Gmail, Calendar, and Drive is what makes this more than a notification list.
  5. Error resilience matters — good skills handle failed data sources gracefully instead of breaking.
  6. Iteration is expected — your first briefing won't be perfect, and that's by design.
