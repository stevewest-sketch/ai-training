Projects as Basic AI Agents

See Projects not just as folders, but as the foundation for building AI agents

The Agent Lens

"Projects are just a little agent. This is the most basic form of an AI agent—a space that has context that permeates across conversations." — Gladly AI Team

From Folder to Agent

In the Beginner path, you learned Projects as persistent workspaces. Now let's reframe them through an agent lens—Projects have persistent memory, specialized knowledge, consistent behavior, and tool access.

Key Concepts

What makes Projects powerful

Persistent Context

Projects remember uploaded files and instructions across conversations, so you don't restart from scratch.

Reference Materials

Add documents, code files, and data that Claude can reference when answering questions or generating content.

Custom Instructions

Set project-specific behavior like tone, format preferences, and domain-specific terminology.

Team Sharing

Share projects with teammates so everyone works with the same context and instructions.

Creating a Project

Step-by-step guide to setting up your first project

1

Open Projects in Claude

Click the Projects icon in the sidebar or go to claude.ai/projects to access your projects.

2

Create New Project

Click "New Project" and give it a descriptive name that reflects its purpose.

3

Add Reference Materials

Upload documents, paste text, or add files that Claude should know about. Think style guides, data files, or documentation.

4

Write Custom Instructions

Tell Claude how to behave in this project. Include role, tone, output preferences, and any domain-specific context.

5

Start a Conversation

Begin chatting with Claude in your project. It now has access to all your reference materials and follows your custom instructions.

Projects vs. Skills

Projects are best for ongoing work with consistent context—like a research project or content creation for a specific product. Skills are better for repeatable actions you trigger on demand—like generating a weekly report or processing data in a specific format.

Project Ideas

Examples of effective Claude Projects

Product Knowledge Base

Upload product docs, FAQs, and positioning guides. Ask Claude questions about the product anytime.

Content Style Guide

Add brand guidelines, voice/tone docs, and example content. Generate on-brand content consistently.

Research Repository

Compile research papers, interview notes, and data. Ask Claude to synthesize findings or find connections.

Code Documentation

Upload README files, API docs, and architecture diagrams. Get help understanding or extending the codebase.
