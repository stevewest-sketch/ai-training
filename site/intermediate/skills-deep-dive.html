Skill Creation Framework

Follow this 6-step process to create effective skills

Step Action Why It Matters
1 Identify the repetitive task Skills save time on tasks you do frequently
2 Document your manual process You can't automate what you can't describe
3 Write a clear description This determines when the skill fires
4 Create step-by-step instructions Be specific about inputs, outputs, and edge cases
5 Test with real examples Validate against actual use cases
6 Iterate on the description Tune until it triggers at the right times

Skill Structure

Skills are markdown files with two critical sections

skill-example.md
# Description "This skill analyzes customer Slack channels weekly and surfaces important discussions, questions, and sentiment patterns for account review." # Instructions 1. Connect to the specified Slack channels 2. Pull messages from the past 7 days 3. Identify key themes, questions, and concerns 4. Generate a summary report with action items 5. Output as markdown or send to Slack channel
Key Insight

Focus most attention on tuning the description sectionâ€”this determines when your skill fires. Claude scans all skill descriptions at the start of each conversation to decide which ones are relevant.

Automation Progression

Skills can evolve from manual triggers to fully automated workflows

Phase Where to Run Use Case
Phase 1 Chat / Cowork Manual trigger, learning the skill
Phase 2 Claude Code Scheduled via cron job
Phase 3 GitHub Actions + Slackbot Fully automated recurring runs
