Why Skills Exist

Claude Code doesn't retain contextual memory between sessions. Skills provide that persistent memory for repeatable processes, so you don't have to re-explain your workflows every time you start a new conversation.

The Matrix Analogy

Skills are like uploading kung fu to Neo in The Matrix—you're giving Claude the knowledge to do a specific repeatable task instantly, without having to explain it every time. Just trigger the skill, and Claude already knows what to do.

Step 1: Understanding Skill Structure

Skills are markdown files with two critical sections that tell Claude when and how to act

skill-example.md
# Description
"This skill analyzes customer Slack channels weekly
and surfaces important discussions, questions, and
sentiment patterns for account review."

# Instructions
1. Connect to the specified Slack channels
2. Pull messages from the past 7 days
3. Identify key themes, questions, and concerns
4. Generate a summary report with action items
5. Output as markdown or send to Slack channel
Key Insight

Focus most attention on tuning the description section—this determines when your skill fires. Claude scans all skill descriptions at the start of each conversation to decide which ones are relevant.

Open Claude Desktop in Chat or Cowork mode. Skills are best developed interactively before moving to automated workflows.

Step 3: Automation Progression

Skills can evolve from manual triggers to fully automated workflows

Phase Where to Run Use Case
Phase 1 Chat / Cowork Manual trigger, learning the skill
Phase 2 Claude Code Scheduled via cron job
Phase 3 GitHub Actions + Slackbot Fully automated recurring runs

Example Skills to Build

Practice by creating skills for these common use cases

  • Weekly Slack channel pulse analysis across customer accounts
  • Turn demo notes into structured follow-up emails
  • Generate changelog summaries from commit history
  • Create meeting prep briefs from calendar and Slack context
Pro Tips from Gladly AI Team

Build the skill yourself from scratch to learn better—don't just copy someone else's. Add output actions to skills like "send a message to Slack channel" or "create a Google Doc" to make them immediately useful.
