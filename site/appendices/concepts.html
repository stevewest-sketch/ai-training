APPENDIX C

Skills vs Automations vs Projects

Understanding when to use each concept and how they work together

Concept Comparison

Each serves a different purpose in your AI workflow

Concept What It Is When to Use Example
Skill Markdown file defining a capability When you want Claude to know how to do something "Analyze Slack channels for sentiment"
Project Persistent context container Ongoing work with consistent context Competitive Intel knowledge base
Automation Skill that runs on schedule/trigger Hands-free recurring tasks Weekly Slack pulse every Monday

The Recommended Progression

Build your capabilities step by step

  • 1

    Start with Projects

    Projects give you flexibility to experiment with persistent context and custom instructions. Use them to prototype agent behavior through conversation before investing in code.

  • 2

    Create Skills

    Once you identify a repeatable workflow that works in a Project, codify it as a skill. Skills are markdown files that define what Claude should do when certain conditions are met.

  • 3

    Add Scheduling for Automations

    When a skill proves valuable and needs to run without you, add a trigger (time-based or event-based) to turn it into an automation.

Key Insight

Don't skip to automations too quickly. Prove value with manual execution first, then automate what works. This approach reduces wasted effort and ensures your automations actually solve real problems.
