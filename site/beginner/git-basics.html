Key Concepts

Core Git terminology you need to understand

Repository (Repo)

A folder that Git tracks. It contains all your project files and the complete history of changes.

Commit

A snapshot of your changes at a point in time. Like a save point you can return to.

Branch

A parallel version of your code. Work on features without affecting the main codebase.

Push / Pull

Push sends your commits to GitHub. Pull downloads others' changes to your machine.

Essential Git Commands

The basic commands you'll use daily

terminal
# Check the status of your changes
git status

# Stage files for commit
git add filename.txt
git add . # stages all changes

# Commit your staged changes
git commit -m "Add new feature"

# Push commits to GitHub
git push

# Pull latest changes from GitHub
git pull
Claude Code + Git

When you use Claude Code, it can handle Git operations for you. You can ask Claude to "commit these changes" or "push to GitHub" and it will execute the commands. Understanding the basics helps you know what Claude is doing.
