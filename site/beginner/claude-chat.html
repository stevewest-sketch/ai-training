Introduction to Claude Chat

Master the conversational AI interface you'll use every day

Learning Objective

By the end of this module, you'll use Claude Chat confidently for brainstorming, drafting, learning, and quick productivity wins.

Overview

In Module 1.2, you learned about Claude's three modes: Chat, Cowork, and Code. Now let's dive deep into Claude Chat — the mode you'll use most frequently.

Claude Chat is where you think together with Claude. It's your always-available thinking partner — ready to brainstorm ideas, draft content, explain concepts, and help you work through problems in conversation.

The Key Difference:
Claude Chat is for thinking together.
Claude Cowork is for working together.

The Mental Model

Claude Chat = A brilliant colleague you can call anytime to talk through ideas
Claude Cowork = That same colleague sitting at your desk, able to actually do the work with you

What Chat Mode Does Best

Chat is optimized for conversational, iterative work where you're actively engaged

✓ Quick Questions & Explanations

Get instant answers, clarifications, and explanations on any topic.

example prompt
What's the difference between ARR and MRR? Explain it simply.

✓ Brainstorming & Ideation

Generate ideas, explore possibilities, and think through problems together.

example prompt
I need 10 creative angles for our Q2 email campaign targeting mid-market retail.

✓ Content Drafting

Write and refine emails, documents, presentations, and more through conversation.

example prompt
Draft a follow-up email to a prospect who went dark after our demo. Make it warm but direct.

✓ Learning & Exploration

Understand new concepts, get context on topics, and build knowledge.

example prompt
Explain how vector embeddings work. I'm technical but new to AI.

Chat Mode Limitations

Understanding these helps you know when to switch to Cowork

✗ Cannot Access Local Files

Chat can't browse your computer. You need to upload files directly, or use Cowork for file system access.

Workaround: Drag-and-drop files into the chat, or use Connectors (next module).

✗ Cannot Modify Files on Your Computer

Chat can create content but can't save it to specific locations on your computer.

Workaround: Copy/export from artifacts, or use Cowork for file operations.

✗ Cannot Run Long Processes Independently

Chat is conversational — it works with you, not for you in the background.

When to switch: If you find yourself saying "do this while I go do something else" → Cowork.

Rule of Thumb

If you find yourself saying "now take that and..." or "can you actually do this?" — you probably want Cowork (covered in the Intermediate path).

Chat in Action: Real Examples

Let's look at how Chat shines in everyday work scenarios

Example 1: Quick Research & Synthesis

You're preparing for a call with a prospect and need quick context on their industry.

What You'd Say:
prompt
I'm calling Sephora's CX team tomorrow. Give me: 1. A quick overview of beauty retail CX trends 2. The 3 biggest challenges their industry faces 3. How competitors are using AI in this space
What Chat Does:
  • Synthesizes relevant information
  • Organizes it in a clear format
  • Gives you conversation-ready talking points

Key Point: This is research and synthesis — Chat's sweet spot.

Example 2: Content Drafting with Iteration

You need to write a customer case study email but want to get the tone right.

What You'd Say:
prompt
Help me write an email to our VP asking if we can publish their logo as a customer reference. Context: - They've been a customer for 8 months - Just renewed with expanded usage - We have a great relationship with the CSM - Never asked for reference permission before Make it warm and appreciative, not pushy.
You Iterate:
follow-up prompt
Good start. Can you: - Make it shorter (2 paragraphs max) - Add a specific mention of their NPS improvement - End with a softer call-to-action

Key Point: The back-and-forth iteration is what makes Chat powerful for drafting.

Example 3: Brainstorming Ideas

You need creative angles for a webinar topic but feel stuck.

What You'd Say:
prompt
I'm planning a webinar on "AI in Customer Experience" for retail brands. Help me brainstorm: - 5 compelling title options - 3 different angles we could take - Discussion questions that would spark engagement Our audience is CX leaders at mid-market and enterprise retail companies.
What Chat Does:
  • Generates diverse options
  • Provides rationale for each
  • Asks follow-up questions to refine

Key Point: Brainstorming is conversational by nature — Chat excels here.

Example 4: Learning a New Topic

You need to understand a technical concept for an upcoming customer conversation.

What You'd Say:
prompt
I need to understand "customer data platforms" (CDPs) for a prospect conversation. I'm not technical. Explain: 1. What a CDP is in simple terms 2. How it's different from a CRM 3. Why a CX team would care about CDP integration 4. Questions I should ask about their CDP stack
What Chat Does:
  • Explains at your level
  • Connects concepts to your context (CX)
  • Prepares you for the real conversation

Key Point: Learning through dialogue helps you understand, not just memorize.

Three Types of Persistent Information

Understanding what Claude remembers and where

Type What It Stores How to Edit Scope
Memory Facts Claude learns from conversations Settings → Memory All conversations
Preferences Default model, language, theme Settings → Preferences All conversations
Project Instructions Custom behavior rules Project Settings That project only

Managing Memory

Add to memory "Remember that I prefer concise responses"
Check memory "What do you remember about me?"
Remove from memory "Forget what you remember about my job title"
View/delete directly Settings → Memory → Click ✕ on any item

Memory vs. Project Instructions

Scenario Use Memory Use Project Instructions
"Always use formal tone with me" ✓
"Use formal tone in this project only" ✓
"I work on the Marketing team" ✓
"This project is for Q1 SKO" ✓
Rule of Thumb

Memory = about YOU (everywhere). Project Instructions = about the WORK (that project only).

Upload vs. Connectors

Use Upload for local files on your computer. Use Connectors when files live in Google Drive, email, Notion, or other cloud services (next module).

Key Takeaways

  1. Chat = conversational collaboration — thinking together, not task execution
  2. Best for: Q&A, brainstorming, drafting, learning, iteration
  3. Limitations: Can't access local files (upload instead) or work independently
  4. Memory persists across conversations — use it for your preferences
  5. When you need file access or background work → Switch to Cowork (Intermediate path)
